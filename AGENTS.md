# AGENTS Guidance (Repository Root)

## Scope
These instructions apply to the full repository unless overridden by a deeper `AGENTS.md`.

## Documentation Standard
For important modules and crate roots, keep documentation concise and production-oriented:
- `README.md`: purpose, contents, usage, notes.
- `SCOPE.md`: in scope, out of scope, sensitive changes, compatibility.
- `ARCHITECTURE.md`: components, data flow, dependencies, boundaries.
- Add `SECURITY.md` and `TESTING.md` where risk or validation complexity justifies them.

## Quality Rules
- Use institutional and precise English.
- Avoid placeholders and ambiguous TODO-style prose.
- Keep MIT-license and liability context explicit in top-level governance docs.
