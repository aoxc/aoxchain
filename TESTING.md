# Testing and Validation Policy

## Validation layers

- **Unit tests:** crate-level logic and invariants.
- **Integration tests:** cross-crate and end-to-end flows (`tests/`).
- **Readiness checks:** operational and release controls generated by CLI workflows.
- **Evidence validation:** script-driven artifact generation and consistency checks.

## Baseline commands

- `cargo test --workspace`
- `cargo clippy --workspace --all-targets --all-features -- -D warnings`
- `cargo fmt --all --check`

## Change-driven requirements

The following classes require targeted regression coverage and documentation updates:
- consensus behavior,
- deterministic execution/state transitions,
- persistence formats and migrations,
- public API contracts,
- key and signing workflows.

## Audit alignment

Validation outputs should be linked to specific artifacts and retained for review. Test and readiness claims without evidence linkage should be treated as incomplete.
