# System Status and Operational Posture

## Status model

AOXChain tracks progress through measurable readiness controls:

- configuration validation,
- profile and baseline parity,
- key operational state,
- release-evidence completeness,
- production-closure artifact completeness.

## Readiness semantics

Readiness scoring is generated from weighted controls and expressed as:
- stage,
- score,
- blockers,
- remediation plan,
- dual-track progress (testnet/mainnet),
- full-surface readiness.

## Evidence sources

- `artifacts/release-evidence/`
- `artifacts/network-production-closure/`
- readiness markdown reports generated by the CLI (`aoxcmd`).

## Intended use

Use the status model for governance and engineering planning. Do not interpret scores as legal certification or a guarantee of production safety.
