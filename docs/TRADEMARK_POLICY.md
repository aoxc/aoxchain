# AOXChain Trademark and Brand Policy

## 1) Purpose

This policy separates code rights from brand rights.

- Code rights are governed by the repository software license.
- Brand rights are controlled separately and are not automatically granted by code license terms.

## 2) Covered marks/assets

This policy applies to project names, logos, and related brand identifiers, including repository brand assets under `logos/`.

## 3) Allowed nominative use

Without separate written permission, third parties may use project names only for factual reference, such as:

- identifying compatibility,
- describing forks,
- citing source origin.

Use must not imply official endorsement, partnership, certification, or governance authority.

## 4) Restricted use

Without explicit written permission, you may not:

- use AOXChain/AOXC marks or logos in a way that implies official status,
- reuse logos/icons for commercial product branding,
- present modified software as official AOXChain releases,
- register confusingly similar names, domains, or package identities.

## 5) Fork naming guidance

Forks should adopt clearly distinct names and visual identities.

- Keep attribution to upstream source.
- Avoid confusing users about governance, trust, or release authority.

## 6) Control-plane and UI representation requirement

Operator UIs, dashboards, or downstream admin tools using AOXC branding must not imply protocol authority beyond what backend controls actually enforce.

## 7) Reservation of rights

All trademark and brand rights are reserved except where explicit written permission is granted.
